# 知识点卡片格式规范

每个知识点卡片包含以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识，如 "kp01" |
| lecture | int | 是 | 所属讲次 |
| title | string | 是 | 知识点名称，与课件术语一致 |
| definition | string | 是 | 概念定义，1-3 句 |
| key_points | string[] | 是 | 关键要点，3-5 条 |
| memory_hook | string | 是 | 记忆钩子，通俗类比/口诀 |
| common_mistakes | string[] | 是 | 常见错误，2-3 条 |
| keywords | string[] | 是 | 题目匹配关键词，5-15 个 |

## 编写原则

1. **定义来源课件**：不用自己的话改写课件原文
2. **关键要点分层**：先概念特征，再分类方法，最后应用场景
3. **记忆钩子要通俗**：类比生活中的事物，或编成顺口溜
4. **常见错误要具体**：不要说"容易混淆"，要说"容易和 XXX 混淆"
5. **关键词要有区分度**：既要覆盖对应题目，又不能太泛（如"创新"匹配太多）

## 示例

```json
{
  "id": "kp18",
  "lecture": 3,
  "title": "Timmons创业过程模型",
  "definition": "Jeffry Timmons于1994年提出，以机会驱动为核心，强调创业过程中机会、团队、资源三要素的动态平衡。",
  "key_points": [
    "三要素：机会、团队、资源",
    "以机会为核心驱动",
    "三要素需动态平衡",
    "创业机会具有模糊性和不确定性"
  ],
  "memory_hook": "Timmons三要素：机（机会）团（团队）资（资源）。",
  "common_mistakes": [
    "记成资金、市场、技术",
    "忽视动态平衡"
  ],
  "keywords": ["Timmons", "创业过程模型", "机会", "团队", "资源", "三要素", "动态平衡"]
}
```
